Как собрать B2B-базу контактов без бана аккаунта: полный гайд по парсингу в 2026

Холодные продажи начинаются с базы контактов. Если база «дохлая» — менеджеры сливают время на автоответчики и неактуальные номера. Если база собрана с нарушением правил площадок — рискуете получить бан аккаунтов. Рассказываем, как собрать живую базу легально и без рисков.

Законно ли это?

Да. Сбор публично доступных данных компаний (название, телефон, email с официального сайта, ИНН из ЕГРЮЛ) не нарушает 152-ФЗ, потому что это данные юридических лиц, а не физических. Мы не собираем персональные данные частных лиц без их согласия. Все источники — из правового поля.

Важно: Не парсим соцсети в обход их API. За это действительно банят. Используем только открытые реестры и карты.

Источники для сбора

  1. 2ГИС и Яндекс Карты — контакты компаний с телефонами и сайтами
  2. Google Maps — международный охват, отзывы, рейтинг
  3. Тендерные площадки (ЕИС) — компании, которые прямо сейчас ищут подрядчиков
  4. Официальные сайты компаний — разделы «Контакты» и «Реквизиты»
  5. ЕГРЮЛ / ФНС — ИНН, ОГРН, юридический адрес, статус компании

Инструменты

  • Python + BeautifulSoup / Scrapy — гибкие кастомные парсеры под любые источники
  • Готовые сервисы — подходят для простых задач, но ограничены по кастомизации
  • Расширения браузера — только для разовых выгрузок, не для системной работы

Анти-бан защита

  • Прокси-серверы: распределённые IP-адреса, имитация поведения обычного пользователя
  • Задержки между запросами: от 1 до 5 секунд, чтобы не триггерить защиту площадок
  • Лимиты: не более 1000 запросов в час с одного IP
  • User-agent: маскируем парсер под обычный браузер
  • Антидетект-браузеры: для сложных задач с авторизацией

Что делать с собранным?

  1. Верификация номеров: автоматический прозвон-сброс (определяет живой номер)
  2. Валидация email: проверка синтаксиса и MX-записи домена
  3. Сверка с ЕГРЮЛ: отсев ликвидированных и недействующих компаний
  4. Выгрузка в CRM: через API AmoCRM/Bitrix24. Новый контакт → сделка → задача менеджеру

Заключение

Парсинг — это не «серая зона» при правильном подходе. Соблюдайте законы, используйте легальные источники и не экономьте на верификации. Живая база окупается за 1-2 недели, потому что менеджеры перестают слушать гудки и начинают продавать.
